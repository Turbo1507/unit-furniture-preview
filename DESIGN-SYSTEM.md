# UNIT.FURNITURE — дизайн-система (v1, 2026-07-16)

Источник истины для отступов/типографики/сетки. Токены живут в `css/styles.css` (`:root`, секция «шкала отступов»). Все последующие правки сайта сверяются с этим документом, а не «на глаз».

## 1. Сетка и контейнер

- `--container: 1280px` — максимальная ширина контента, `.container { padding: 0 40px }` (0 20px на мобилке ≤720px).
- Колоночные сетки: 4 кол. (dir-grid, mat-strip), 3 кол. (catgrid), 2 кол. (sol-grid, custom-hub боковые), провал в 2 → 1 колонку на ≤1024px → 720px по компонентам с фото (не оставляем «сиротский» 2-колоночный шаг для номерных списков типа roadmap — там сразу 1 колонка на ≤1024px).
- Каталог товаров: 3 колонки → 2 (≤1024) → 1 (≤720).

## 2. Шкала отступов (макетный ритм)

Переменные в `:root`:

| Токен | px | Роль |
|---|---|---|
| `--sp-2xs` | 8 | между тесно связанными строками (тег + текст) |
| `--sp-xs` | 12 | внутри мелких групп (чипы, теги, check-grid gap) |
| `--sp-sm` | 16 | между полями формы, мелкими элементами, dir-grid/catgrid gap на мобилке |
| `--sp-md` | 24 | между карточками в сетке (gap грида) |
| `--sp-lg` | 32 | под CTA сразу после короткого текста (hero-actions) |
| `--sp-xl` | 40 | после интро-параграфа перед CTA-кнопками; фильтры; gap между независимыми группами полей |
| `--sp-2xl` | 56 | интро секции (h2[+lead]) → контентный грид. **Правило: всегда 56, даже если lead-параграфа нет** (см. фикс check-grid) |
| `--sp-3xl` | 64 | между независимыми смысловыми блоками одной секции (например .sol-row) |
| `--sp-4xl` | 120 | `.section { padding: 120px 0 }` — ритм между секциями. 72px на ≤720px |

Правило по умолчанию: **любая новая раскладка берёт значение из этой шкалы**. Значения вне шкалы (2,3,4,5,6,7,9,10,11,13,14,18,22,26,28,36,66,72,73,78,88px и т.д.) допустимы только для **компонентных деталей** — border-radius мелких элементов, паддинг иконок/пилюль, координаты декоративных SVG, — то есть там, где решает не ритм страницы, а форма конкретного виджета.

## 3. Типографика

- Шрифт: Helvetica Neue / Helvetica / Arial.
- `h1`: `clamp(40px, 5.2vw, 72px)`, вес 700, line-height 1.15, letter-spacing -0.01em.
- `h2`: `clamp(30px, 3.6vw, 48px)`, те же вес/line-height/tracking.
- `h3`: 20px, вес 600.
- Основной текст (лид-параграфы, 16–18px): line-height 1.55–1.6.
- Мелкий текст в карточках/тегах (12–14px): line-height 1.45–1.5.
- Длинный связный текст (юр.страницы, 15px): line-height 1.7 — читаемость важнее компактности.
- Правило: line-height ОБРАТНО пропорционален размеру шрифта и ПРЯМО пропорционален длине связного текста — не унифицируем в одно число, но держим строго в одном из трёх диапазонов выше в зависимости от роли текста.

## 4. Скругления и тени

- `--r-sm: 10px` (кнопки, инпуты, мелкие пилюли-детали).
- `--r-md/--r-lg: 15px` (карточки, крупные панели, фото-блоки).
- `--r-pill: 999px` (таб-пилюли, бейджи).
- `--shadow-sm` — карточки категорий/решений на фото-подложке.
- `--shadow` — крупные фото-панели (band, final-photo, ch-center).
- Карточки каталога (`.product`) — отдельная, более мягкая тень (см. п.12 фидбека Босса), не путать с `--shadow`/`--shadow-sm`.

## 5. Цвет

- `--white #FFFFFF`, `--graphite #3A3A3A`, `--stone #F0F0F0`, `--interlace #BEBEBC`, `--blue #1B1FEA` — из брендбука «Креаторам.pdf», не менять произвольно.
- Синий — только акцент (кнопки, ссылки, лого «UNIT», активные состояния), не заливка больших площадей.

## 6. Правило секции

Каждая содержательная секция страницы (кроме hero и band-панелей) — это:

```
<section class="section [section-gray]">
  <div class="container">
    <h2>…</h2>
    <p class="section-lead">…</p>   <!-- необязательно -->
    <div class="[контент-грид]">…</div>  <!-- margin-top: var(--sp-2xl), т.е. 56px -->
  </div>
</section>
```

`.section` даёt 120px сверху/снизу (72px на мобилке). Первая секция после `.page-hero` — класс `.first-after-hero` (56px сверху, 40px на мобилке) вместо полных 120px, чтобы не было двойного воздуха под шапкой страницы.

## 7. Мобильные правила

- Хиро (главная) — full-bleed фото, хедер прозрачный до скролла/до открытия меню.
- Закреплённая CTA-планка снизу (66px) — учитывается в расчёте высоты hero (`calc(100svh - 66px)`) и в `body{padding-bottom}`.
- Роадмапы (4/5/6 шагов) — вертикально одной колонкой уже с ≤1024px, без промежуточного 2-колоночного шага.
- Фильтры каталога — вертикальным стеком, оба дропдауна на всю ширину.
